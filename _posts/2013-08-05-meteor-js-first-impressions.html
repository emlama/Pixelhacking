Since taking up Node.js development I've only worked with Express.js, a more traditionally styled framework. All-in-all it has provided all the functionality that I would have expected to find in a light-weight framework (see: Sintra and Flask).

But this blog post isn't about that. This is about Meteor and a new approach to building web applications.

The primary goal of Meteor (and others, Derby.js/Sails.js) is to move application development into a new era by eliminating the traditional REST model and moving towards real-time data synchronization between the client (browser) and server.

At work we've recently started in on an internal product that requires multiple users interacting real-time and decided to build a Meteor.js version of the product in parallel with the actual product to see how capable the framework really is.

To date I've been delighted with both the framework and the community. I would say the framework is getting close to 'production ready' for new and upcoming products but your mileage my vary.
